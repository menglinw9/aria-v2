# Week 4 Homework - ARIA v2.0

## 專案概述

本專案為遙測與空間分析 Week 4 的作業，主要目標是結合向量與網格資料進行避難所洪災風險評估。

## 資料來源

- **河川資料**: 水利署河川多邊形資料 (`../../week3/homework/data/riverpoly.shp`)
- **避難所資料**: 消防署避難收容處所資料 (`../../week3/homework/data/避難收容處所.csv`)
- **鄉鎮界線**: 國土測繪中心鄉鎮市區界線
- **DEM 資料**: `data/clipped_dem.tif` (花蓮縣數值高程模型)

## 分析目標縣市

- **花蓮縣** (透過 `.env` 檔案中的 `TARGET_COUNTY` 參數設定)，DEM 資料僅涵蓋花蓮縣範圍
- 河川距離計算使用歐幾里得距離
- 風險分級基於地形因子，未考慮降雨強度等因素
- 所有輸出檔案使用 UTF-8 編碼

## 分析流程

1. **資料載入與清理**
   - 載入河川、避難所、鄉鎮界線資料
   - 統一座標系統為 EPSG:3826 (TWD97)
   - 篩選目標縣市內的避難所

2. **地形分析**
   - 從 DEM 計算坡度（度）
   - 為每個避難所建立 500m 緩衝區
   - 計算平均高程、最大坡度、高程標準差

3. **風險評估**
   - 計算避難所到最近河川的距離
   - 依據坡度、高程、河川距離進行風險分級
   - 風險等級：極高風險、高風險、中風險、低風險

4. **結果輸出**
   - `terrain_risk_audit.json`: 詳細風險分析結果
   - `terrain_risk_map.png`: 視覺化地圖與統計圖

### AI 診斷日誌
- .env 檔案中設定 TARGET_COUNTY=花蓮縣，但程式讀取到系統環境變數為臺北市，可能是因為先前作業的.env是設定台北市因此互相干擾。利用load_dotenv(override=True)強制覆蓋現有環境變數。
- zonal_stats 返回 NaN 值，導致後續分析失敗。經過檢查發現是因為CRS 坐標系統不一致，設定讓所有座標系統都強制與dem (crs=3826)相同。
